Хранилище конфигурации 1С

1. Что такое хранилище конфигурации?

Хранилище конфигурации — это централизованная база данных для хранения версий конфигурации и организации коллективной разработки. Оно позволяет нескольким разработчикам работать над одним проектом одновременно, контролируя изменения и предотвращая конфликты.

Коллективная разработка

Одновременная работа нескольких разработчиков над одной конфигурацией с контролем изменений

История изменений

Полное отслеживание всех изменений конфигурации с возможностью отката к любой версии

Ветвление и слияние

Создание веток разработки для новых функций с последующим слиянием в основную ветку

Контроль доступа

Разграничение прав разработчиков на чтение и изменение различных объектов конфигурации

2. Архитектура и рабочий процесс

Рабочий процесс с хранилищем

Загрузка последней версии

Получение актуальной конфигурации из хранилища перед началом работы

Работа с изменениями

Редактирование конфигурации в режиме блокировки объектов

Выгрузка изменений

Сохранение изменений в хранилище с комментариями и освобождение объектов

4. Разрешение конфликтов

Типичные конфликты и их решение

Конфликты возникают, когда несколько разработчиков одновременно изменяют один и тот же объект конфигурации. Хранилище предлагает различные стратегии разрешения конфликтов:

  • Автоматическое слияние: Для изменений в разных частях объекта
  • Ручное слияние: При одновременном изменении одного свойства
  • Выбор версии: Принятие изменений одного разработчика с отклонением других
  • Объединение изменений: Интеграция полезных изменений из всех версий

Важно: Регулярно обновляйте конфигурацию из хранилища перед началом работы и своевременно выгружайте изменения после завершения задачи. Это минимизирует риск конфликтов.

5. Лучшие практики работы с хранилищем

  • Всегда делайте закладку на задачу перед началом работы с объектами
  • Пишите подробные комментарии при выгрузке изменений
  • Не держите объекты захваченными дольше необходимого
  • Регулярно создавайте версии конфигурации по завершении этапов
  • Используйте ветвление для разработки крупных функциональностей
  • Перед выпуском в production создавайте релизную версию

Рекомендация: Настройте автоматическое создание версий при каждом успешном прохождении тестов и используйте систему непрерывной интеграции для контроля качества.

Контрольные вопросы

Вопрос 1: Что такое хранилище конфигурации и для чего оно предназначено?

Хранилище конфигурации — это централизованная система управления версиями для конфигураций 1С, предназначенная для организации коллективной разработки, контроля изменений, хранения истории модификаций и предотвращения конфликтов при одновременной работе нескольких программистов.

Вопрос 2: Какие основные операции поддерживаются при работе с хранилищем?

Основные операции включают подключение к хранилищу, загрузку последней версии конфигурации, захват объектов для редактирования, выгрузку изменений, создание версий, сравнение версий, разрешение конфликтов и управление правами доступа разработчиков.

Вопрос 3: Как организовать совместную разработку с использованием хранилища?

Совместная разработка организуется через выделение объектов разработчикам с системой блокировок: разработчик захватывает объекты для редактирования, вносит изменения, а затем выгружает их обратно в хранилище, освобождая объекты для других.

Вопрос 4: Что такое версия конфигурации и как происходит управление версиями?

Версия конфигурации — это фиксированное состояние всех объектов конфигурации в определенный момент времени; управление версиями включает создание версий по завершении этапов разработки, их маркировку, хранение истории и возможность отката к любой предыдущей версии.

Вопрос 5: Как разрешать конфликты при совместной разработке?

Конфликты разрешаются через механизм сравнения версий и слияния изменений: система предлагает варианты автоматического слияния, ручного объединения изменений или выбора одной из версий; для сложных конфликтов требуется вмешательство ведущего разработчика.

Вопрос 6: Какие настройки безопасности применяются к хранилищу конфигурации?

Настройки безопасности включают аутентификацию пользователей, разграничение прав доступа к объектам конфигурации, контроль операций (чтение, изменение, удаление), ведение журнала действий и настройку политик паролей для доступа к хранилищу.

Упражнения для самостоятельной работы:

  1. Настройте хранилище конфигурации для существующей базы
  2. Отработайте процесс загрузки и выгрузки изменений
  3. Создайте и примените версию конфигурации

Инструкция по выполнению (упражнение 1):

Настройка хранилища в конфигураторе

Шаг 1: Откройте конфигуратор

Запустите конфигуратор вашей базы данных с правами администратора

Шаг 2: Создание хранилища

В меню выберите: Хранилище конфигурации → Создать хранилище...

Укажите путь к папке, где будет размещено хранилище (рекомендуется использовать отдельный диск или сетевую папку)

Шаг 3: Настройка параметров

Задайте настройки безопасности:

- Создайте пользователя-администратора хранилища

- Установите сложный пароль

- Настройте политику блокировок (рекомендуется "Автоматическая блокировка изменяемых объектов")

Шаг 4: Начальная выгрузка

После создания хранилища выполните начальную выгрузку конфигурации:

Выберите: Хранилище конфигурации → Выгрузить конфигурацию в хранилище

Введите комментарий (например, "Начальная версия") и подтвердите выгрузку

Шаг 5: Настройка прав доступа

Добавьте пользователей для коллективной разработки:

Перейдите в Хранилище конфигурации → Пользователи хранилища

Добавьте новых пользователей и назначьте им права (разработчик, читатель, администратор)

Инструкция по выполнению (упражнение 2):

Процесс работы с изменениями

Шаг 1: Подключение к хранилищу

Откройте конфигуратор базы и выберите: Хранилище конфигурации → Подключиться к хранилищу...

Укажите путь к хранилищу и введите свои учетные данные

Шаг 2: Загрузка последней версии

Перед началом работы загрузите актуальную конфигурацию:

Выберите: Хранилище конфигурации → Загрузить конфигурацию из хранилища

Подтвердите замену текущей конфигурации

Шаг 3: Получение объектов для редактирования

Откройте список объектов конфигурации (F2)

Выделите нужные объекты и выберите: Хранилище конфигурации → Получить для изменения

Объекты будут заблокированы для других разработчиков

Шаг 4: Внесение изменений

Отредактируйте объекты конфигурации

Сохраните изменения (F7)

Шаг 5: Выгрузка изменений

После завершения работы выгрузите изменения:

Выберите: Хранилище конфигурации → Выгрузить изменения в хранилище

Введите подробный комментарий о внесенных изменениях

Объекты будут освобождены для других разработчиков

Инструкция по выполнению (упражнение 3):

Управление версиями

Шаг 1: Создание версии

Убедитесь, что все изменения выгружены в хранилище

В конфигураторе выберите: Хранилище конфигурации → Создать версию...

Укажите номер версии (например, "1.0.0") и описание изменений

Шаг 2: Просмотр истории версий

Для просмотра созданных версий выберите:

Хранилище конфигурации → История хранилища...

Вы увидите список всех версий с датами, авторами и комментариями

Шаг 3: Загрузка конкретной версии

Для загрузки определенной версии в конфигурацию:

В истории хранилища выделите нужную версию и выберите "Загрузить"

Подтвердите замену текущей конфигурации

Шаг 4: Сравнение версий

Для анализа различий между версиями:

Выберите две версии в истории и нажмите "Сравнить"

Конфигуратор покажет все изменения между выбранными версиями

Шаг 5: Откат к предыдущей версии

В случае необходимости отката:

Загрузите нужную версию из хранилища

Выгрузите ее как текущее состояние конфигурации

Создайте новую версию с комментарием об откате

Примечание: Все операции с хранилищем должны выполняться с правами соответствующего уровня доступа. Административные операции (создание хранилища, управление пользователями) требуют прав администратора хранилища.
