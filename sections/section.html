Структура конфигурации и проведение документов

1. Архитектура конфигурации 1С

Конфигурация в 1С представляет собой иерархическую структуру объектов метаданных, которые определяют логику, данные и интерфейс прикладного решения. Понимание этой структуры критически важно для эффективной разработки.

Уровни конфигурации

Уровень данных

Справочники, Документы, Регистры

Уровень логики

Модули объектов, Общие модули, Обработки

Уровень интерфейса

Формы, Отчеты, Панели навигации

2. Основные объекты конфигурации

Справочники

Списки однотипных элементов (контрагенты, номенклатура, сотрудники)

Документы

Объекты для фиксации хозяйственных операций с механизмом проведения

Регистры

Хранение периодических данных: остатки, обороты, сведения

Отчеты

Средства анализа и вывода данных в структурированном виде

Обработки

Вспомогательные программы для выполнения специфических операций

Планы видов характеристик

Хранение дополнительных свойств объектов в разрезе видов характеристик

3. Механизм проведения документов

Проведение документа — это ключевой механизм, обеспечивающий двойную запись в учетных системах. При проведении документ формирует движения по регистрам, которые обновляют остатки и позволяют строить актуальные отчеты.

Принцип проведения: Каждый документ может быть проведен (отражен в учете) или не проведен (черновик). Отмена проведения (сторнирование) возвращает систему к состоянию до операции.

4. Модули объектов и менеджеров

Каждый объект конфигурации имеет связанные модули:

  • Модуль объекта: Содержит логику, специфичную для конкретного экземпляра объекта (например, обработка событий формы)
  • Модуль менеджера: Содержит общие методы для работы с объектами данного типа (создание, поиск, выборка)

5. Подсистемы и навигация

Подсистемы используются для логической группировки объектов конфигурации и построения навигации в интерфейсе. Они позволяют организовать сложную конфигурацию в понятную для пользователя структуру.

Контрольные вопросы

Вопрос 1: Из каких основных объектов состоит типичная конфигурация 1С?

Типичная конфигурация состоит из справочников (хранение данных), документов (фиксация операций), регистров (хранение периодических данных), отчетов (анализ данных), обработок (вспомогательные операции) и подсистем для организации навигации.

Вопрос 2: Как связаны между собой документы и регистры накопления?

Документы являются регистраторами для регистров накопления — при проведении документа он формирует движения по регистрам, которые изменяют остатки; эта связь обеспечивает актуальность учетных данных и возможность построения отчетов.

Вопрос 3: Что такое модуль объекта и модуль менеджера?

Модуль объекта содержит логику работы с конкретным экземпляром объекта, а модуль менеджера предоставляет общие методы для работы со всеми объектами данного типа, такие как создание, поиск и массовые операции.

Вопрос 4: Как организовать проведение документа с формированием движений?

Проведение организуется через процедуру обработки проведения в модуле документа, где формируются движения по регистрам; при отмене проведения движения удаляются, что возвращает систему в предыдущее состояние.

Вопрос 5: Что такое подсистемы и для чего они используются?

Подсистемы — это логические группы объектов конфигурации, используемые для организации интерфейса пользователя; они позволяют структурировать функционал, создавать панели навигации и управлять видимостью объектов для разных ролей.

Упражнения для самостоятельной работы:

  1. Создайте простую конфигурацию "Торговля"
  2. Реализуйте проведение документа "ПриходнаяНакладная"
  3. Настройте формы списков и элементов для справочников

Пример выполнения (упражнение 2):

// Модуль документа "ПриходнаяНакладная"

// Процедура проведения документа
Процедура ОбработкаПроведения(Отказ, Режим)

    // Очищаем предыдущие движения
    Движения.ОприходованиеТоваров.Записать();
    Движения.ОприходованиеТоваров.Очистить();

    // Обходим все строки табличной части
    Для Каждого СтрокаТовары Из Товары Цикл

     // Формируем движение по регистру накопления "ОстаткиТоваров"
     Движение = Движения.ОприходованиеТоваров.Добавить();
     Движение.Период = Дата;
     Движение.Номенклатура = СтрокаТовары.Номенклатура;
     Движение.Склад = Склад;
     Движение.Количество = СтрокаТовары.Количество;
     Движение.Сумма = СтрокаТовары.Сумма;

    КонецЦикла;

КонецПроцедуры

// Процедура отмены проведения
Процедура ОбработкаОтменыПроведения(Отказ, Режим)

    // Удаляем движения при отмене проведения
    Движения.ОприходованиеТоваров.Записать();
    Движения.ОприходованиеТоваров.Очистить();

КонецПроцедуры

// Функция заполнения цены и суммы по номенклатуре
Функция ЗаполнитьЦены()

    Для Каждого СтрокаТовары Из Товары Цикл

     Цена = УтилитыРаботыСДанными.ПолучитьЦенуНоменклатуры(
     СтрокаТовары.Номенклатура,
     Дата
     );

     СтрокаТовары.Цена = Цена;
     СтрокаТовары.Сумма = Цена * СтрокаТовары.Количество;

    КонецЦикла;
